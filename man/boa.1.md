# NAME

`boa` - A tool for automating the generation of Debian changelogs from `CHANGELOG.md` files.

# SYNOPSIS

`boa` [_options_] ARGS..

# DESCRIPTION

`boa` is a tool that automates the process of constructing a Debian changelog from a `CHANGELOG.md` file generated by the `auto-changelog` tool. It is designed to be used in a CI/CD pipeline to streamline the generation of the `debian/changelog` file required for Debian packaging.

The structure of the `CHANGELOG.md` file is expected to follow the format produced by `auto-changelog`, which includes version headers, release dates, and categorized lists of changes (e.g., Added, Changed, Fixed). `boa` parses this structure and converts it into the format required by Debian packaging tools. The generated `debian/changelog` file will include entries for each version, formatted according to Debian standards, and will include the package name, release type, urgency, and maintainer information.

To keep the both the `CHANGELOG.md` and `debian/changelog` files from being populated with redundant information or commits that do not effect the acctual package, it is recommended to use the `--commit-limit` option of `auto-changelog` to exclude certain commits from being included in the generated changelog. See the `AUTO-CHANGELOG` section for more information.

# OPTIONS

`-i`, `--input` <file>
:   Path to the input `CHANGELOG.md` file (default: `CHANGELOG.md`).

`-o`, `--output` <file>
:   Path to the output `debian/changelog` file (default: `debian/changelog`).

`-p`, `--package` <name>
:   Package name to use in the changelog entries (default: `my-package`).

`-r`, `--release` <type>
:   Release type to use in the changelog entries (default: `unstable`).

`-u`, `--urgency` <urgency>
:   Urgency to use in the changelog entries (default: `medium`).

`-m`, `--maintainer` <name>
:   Maintainer name and email to use in the changelog entries (default: from git config or `DEBEMAIL`/`DEBFULLNAME` environment variables).

`-h`, `--help`
:   Show this help message and exit.

`-v`, `--version`
:   Show version information and exit.

# AUTO-CHANGELOG

`auto-changelog` can be configured to exclude certain commits from being included in the generated `CHANGELOG.md` file using the `--commit-limit` option or by the _.auto-changelog_ json configuration file. This can be useful to prevent commits that do not affect the actual package (e.g., documentation updates, formatting changes, etc.) from being included in the changelog.

---

  ```bash
  auto-changelog --commit-limit "docs/*,formatting/*"
  ```

---

  ```json
  {
    "commitPattern": "^(feat:|fix:|revert:|docs:|bump:|BREAKING CHANGE:)",
  }
  ```

By configuring `auto-changelog` to exclude certain commits, you can ensure that the generated `CHANGELOG.md` file (and consequently the `debian/changelog` file generated by `boa`) only includes relevant changes that affect the package, resulting in a cleaner and more meaningful changelog for users and maintainers.

## AUTO-CHANGELOG CONFIGURATION

`-o`, `--output` <file>
:   Output file, default: CHANGELOG.md (.auto-changelog = "output": "CHANGELOG.md")

`--hide-empty-releases`
:   Hide releases with no commits, default: false (.auto-changelog = "hideEmptyReleases": false)

Run `auto-changelog --help` for more options and information. Any options that has a _-_ between words can be used in the _.auto-changelog_ json configuration file by converting the _-_ to camelCase. For example, `--hide-empty-releases` becomes `hideEmptyReleases` in the json configuration.

Any option that does not take an argument can be used in the json configuration file by setting it to `true`. For example, `--hide-empty-releases` can be set in the json configuration file as `"hideEmptyReleases": true`.
